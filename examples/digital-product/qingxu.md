## Product brief and business goal

清序 Qingxu 是面向团队的 AI 研究工作台，帮助用户收集来源、综合发现并分享清楚的研究叙事。落地页需要表达能力，但不能显得未来主义、过载或历史化。

两个版本现在都是完整的移动端建站方案，而不是单个展示卡片。

## Ordinary AI default proposal

普通 AI 默认方案使用与 SongStyle 改写完全相同的生成提示词、品牌、标题、说明、场景图片、功能信息、行动与业务目标。它仍然专业、可用，但更像常见 SaaS 默认落地页：标题、试用按钮、场景图和功能卡片都出现了，只是研究流程没有形成清楚的阅读顺序。

## Default diagnosis

相同功能被平铺为同等权重，用户可以看到能力列表，却难以判断“来源、综合、叙事”之间的先后关系。问题不是风格完全错误，而是缺少 SongStyle 对必要密度和信息距离的处理。

## UI/UX 执行模型再生成

这次按新的执行模型重新生成两个移动网站方案。普通版与 SongStyle 版使用同一基础生成提示词；Base Prompt 只交代任务和内容，不包含排版、布局、字体、风格、留白或色彩方向；SongStyle 版本只多一句调用 SongStyle Skill 或 Prompt 的要求，所以比较对象不是“科技风 vs 宋式风”，而是同一现代产品是否经过宋代美学项目的执行层处理。

- **Scenario fit / 场景匹配**：AI 研究工作台产品落地页，核心任务是解释来源、综合、分享叙事和团队复核。
- **Page pattern / 页面模式**：普通 AI 默认图使用 standard SaaS landing + image + equal feature cards；SongStyle 在同一模式内把近景任务、中景证据关系、远景团队叙事重新排序。
- **Interaction states / 交互状态**：开始试用、来源查看、团队复核等行动需要 focus、hover、active、disabled、loading、error/success 状态说明。
- **Pre-delivery checks / 交付前检查**：检查 reduced-motion、mobile touch、breakpoint、图片裁切、CTA 可见性和中文标题换行，确认信息密集但不拥挤。

## Controlled generation prompts

### Base Prompt

```text
请为移动端生成一个 AI 研究工作台产品落地页。
品牌：清序 Qingxu。
定位：AI 研究工作台。
标题：从来源到叙事，让研究过程清楚可见。
说明：收集可信来源，综合带引用的发现，并与团队共同整理一条可以被理解和复核的研究叙事。
功能：收集来源、综合发现、分享叙事。
行动：开始试用。
业务目标：让 AI 研究产品显得可信、安静、可复核，并适合真实团队协作。
图片：使用同一张研究工作场景编辑摄影。
```

## SongStyle decisions

SongStyle 版本把“三远法”转译为研究信息架构：近景是当前任务，中景是来源与发现关系，远景是团队共同理解的叙事。

- **删减**：不删减必要功能，只移除不承担解释任务的视觉强调。
- **层级**：让同一组“收集、综合、分享”形成清楚研究叙事。
- **空间**：让每个阶段有阅读停顿，同时保留理解产品所需密度。
- **色彩**：用墨色、雾灰与青瓷色表达现代技术的平静可信。
- **图片**：复用同一张研究场景图片，通过安静裁切支持产品理解。

## 审查结果

SongStyle 改写没有获得更好的文案、图片或初始 Brief。它只多了最后一句 SongStyle 指令，但通过秩序、必要密度与温润表达，让相同的现代技术能力更容易理解和信任。

## Implementation notes

案例保留界面信息密度，并使用语义结构、清楚对比和可访问行动。
