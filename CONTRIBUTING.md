# Contributing

## Before You Contribute

SongStyle is a modern design translation informed by Song-dynasty aesthetics.
Do not present modern interpretations, design rules, or implementation choices
as historical facts.

## Repository Modules And Licenses

Code, build utilities, Design Tokens, prompts, skills, and reference UI use the
MIT License. Documentation, examples, and visual case-study content use CC BY
4.0. Contributions are submitted under the license applicable to their module.

## Cultural And Design Guardrails

Contributions should express restraint, order, warmth, and meaningful negative
space without reducing usability. Avoid treating ink wash, seals, traditional
patterns, or calligraphy as automatic evidence of SongStyle.

## Development Setup

Use Node.js 24 and install dependencies with `npm install`.

## Verification

Run `npm run verify` before submitting a change.

## Pull Request Expectations

Keep changes focused, explain the design reasoning, include relevant tests, and
identify any generated Design Token output.
