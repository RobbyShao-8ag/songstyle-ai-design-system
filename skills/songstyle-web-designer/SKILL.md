# SongStyle Web Designer

Design modern Web experiences that know when to stop. Preserve necessary information and let cultural character come from order rather than symbols.

## Install With Your AI Agent

If your AI Agent supports installing skills from GitHub, copy this sentence into it. 复制这句话：


## Use And Boundaries

Use for websites, landing pages, product interfaces, and written design proposals. Do not use as a shortcut for historical reconstruction, ancient-style decoration, or removing information required for usability, accessibility, truthfulness, or business goals.

Read [SongStyle principles](references/songstyle-principles.md), the [review model](references/review-model.md), and the [UI/UX execution model](references/ui-ux-execution-model.md) before designing.

## Required Inputs

- Product, page goal, audience, and primary user task
- Required content and actions
- Brand and technical constraints
- Existing assets or page problems when redesigning

State assumptions when inputs are incomplete.

## Workflow

1. Clarify the page goal, audience, user task, and required content.
2. Run a Hard-constraint checkpoint: required content, accessibility, truthfulness, technical constraints, business goal, and Chinese heading readability must survive.
3. Write Scenario fit / 场景匹配: identify the product, transaction, data tool, content, ecommerce image, or other scene type and the necessary-density floor.
4. Choose Page pattern / 页面模式: use the smallest useful structure such as hero + evidence + CTA, comparison + CTA, pricing + FAQ, dashboard task flow, search first, or product intro image.
5. Classify information into near, middle, and far layers.
6. Remove, defer, merge, preserve, or clarify information. Explain each meaningful decision with a decision record.
7. Establish hierarchy, rhythm, and functional negative space.
8. Define Interaction states / 交互状态 for focus, hover, active, disabled, loading, and error/success.
9. Select expression strength and apply design foundations without relying on cultural shortcuts.
10. Run Pre-delivery checks / 交付前检查 for reduced-motion, mobile touch, breakpoint behavior, accessible contrast, and link or CTA clarity.
11. Stop adding visual elements when no user, business, accessibility, or meaning function is served.
12. Self-review against every shared review dimension and cite evidence.

## Output Format

1. Page goal and assumptions
2. Hard-constraint checkpoint
3. Scenario fit / 场景匹配
4. Page pattern / 页面模式
5. Near, middle, and far information hierarchy
6. Decision records for removal, deferral, merge, preservation, or clarification
7. Layout and functional negative space
8. Visual direction
9. Interaction states / 交互状态
10. Implementation constraints
11. Pre-delivery checks / 交付前检查
12. Self-review with evidence, risks, and unresolved questions
