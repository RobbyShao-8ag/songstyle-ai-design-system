# 从 Brief 到宋式 Web 设计

1. 明确页面目标与用户任务。
2. 做硬约束检查：确认合规、安全、可访问性、业务流程和关键用户任务不能被宋式表达削弱。
3. 列出必要信息，并标注哪些信息必须保留、哪些需要澄清。
4. 建立近景、中景、远景的信息层级：近景承接当前任务，中景解释结构与证据，远景提供背景、信任和延伸阅读。
5. 对每组内容做删除、延后、合并、保留或澄清决策，记录依据。
6. 建立层级与节奏。
7. 分配功能性留白。
8. 选择表达强度。
9. 应用[设计基础](../../foundations/)。
10. 使用[共享审查清单](../../checklist/)复核。

设计过程必须记录假设与删减决策。不能为了留白删除必要信息，也不能为了文化气质堆叠装饰符号。
