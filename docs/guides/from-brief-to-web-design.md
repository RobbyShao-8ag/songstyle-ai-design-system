# 从 Brief 到宋式 Web 设计

1. 明确页面目标与用户任务。
2. 列出必要信息。
3. 移除、延后或合并非必要信息。
4. 建立层级与节奏。
5. 分配功能性留白。
6. 选择表达强度。
7. 应用[设计基础](../../foundations/)。
8. 使用[共享审查清单](../../checklist/)复核。

设计过程必须记录假设与删减决策。不能为了留白删除必要信息，也不能为了文化气质堆叠装饰符号。
