# 从 Brief 到宋式 Web 设计

1. 明确页面目标与用户任务。
2. 做[硬约束检查](../../principles/priority-and-tradeoffs/)：确认合规、安全、可访问性、业务流程和关键用户任务不能被宋式表达削弱。
3. 用 [UI/UX 执行模型](../ui-ux-execution-model/)完成 Scenario fit / 场景匹配 与 Page pattern / 页面模式，确认这是产品落地页、交易页、数据工具、内容页、电商介绍图片或其他任务场景。
4. 按[必要密度边界](../necessary-density-boundary/)列出必要信息，并标注哪些信息必须保留、哪些需要澄清。
5. 建立[近景、中景、远景](../../foundations/information-distance/)的信息层级：近景承接当前任务，中景解释结构与证据，远景提供背景、信任和延伸阅读。
6. 对每组内容做删除、延后、合并、保留或澄清决策，记录依据。
7. 建立层级与节奏。
8. 分配功能性留白。
9. 定义 Interaction states / 交互状态：focus、hover、active、disabled、loading、error/success。
10. 选择表达强度并应用[设计基础](../../foundations/)。
11. 做 Pre-delivery checks / 交付前检查：reduced-motion、mobile touch、breakpoint、可访问对比度、CTA 与链接路径。
12. 使用[共享审查清单](../../checklist/)复核。

设计过程必须记录假设与删减决策。不能为了留白删除必要信息，也不能为了文化气质堆叠装饰符号。
