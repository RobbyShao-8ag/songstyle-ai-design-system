# 必要密度边界

必要密度指完成任务所需的信息下界。SongStyle 可以通过秩序、节奏和留白降低认知噪声，但不能隐藏必要信息，也不能把业务、法律、价格、风险或专家判断所需内容删掉。

## 页面类型下界

- 产品落地页：必须保留目标用户、核心价值、关键功能、证据、限制、价格或转化路径。可以弱化噪声型营销话术，但不能删除用户判断价值所需的信息。
- 数据或专家工具：必须保留指标定义、单位、筛选条件、异常状态、来源、时间范围和操作反馈。界面可以克制，但专业判断所需密度不能被压低。
- 交易页面：必须保留商品或服务明细、价格、费用、交付条件、风险提示、确认动作和售后信息。不能为了留白弱化成本、责任或最终提交状态。
- 内容型页面：必须保留标题、作者或来源、发布时间、摘要、正文结构和延伸路径。可以调整阅读节奏，但不能让事实来源模糊。

## 决策类型

- 删除：只删除重复、失效、误导或与任务无关的信息。
- 延后：把低频但有价值的信息放入后续区块、详情页或展开层。
- 合并：把相近证据、说明或操作归组，减少重复路径。
- 保留：对任务、合规、信任或专家判断必要的信息必须保留。
- 澄清：当信息是否必要无法判断时，先补充业务目标、用户场景和风险边界。

## 审查问题

- 当前页面类型的必要密度下界是否被明确写出？
- 是否有任何必要信息因为追求留白、克制或宋式气质被删除？
- 产品落地页是否仍然能支持价值判断和转化决策？
- 数据或专家工具是否仍然支持准确判断、追溯和纠错？
- 交易页面是否仍然清楚呈现价格、风险、确认动作和责任边界？
- 每个删除、延后、合并、保留或澄清决定是否说明了依据？
