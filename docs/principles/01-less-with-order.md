# 少而有序 / Less, With Order

`less-with-order`

减少信息不是目的，建立清晰而稳定的关系才是目的。

## 这意味着什么

宋式设计不把“少”当作视觉姿态。页面保留多少内容，应由用户任务、业务目标与信息关系决定。

## 为什么重要

AI 很容易把每条输入扩写成一个模块。没有秩序的丰富会使用户更难判断重点，也会削弱真正重要的信息。

## 如何转译为现代 Web 设计

- 在产品首页中，把核心价值、主要行动与必要信任信息放在稳定阅读路径中，而不是并列展示十个卖点。
- 在复杂工作台中，保留筛选器、状态和引用，但通过分组、对齐与层级减少认知竞争。

## 常见误区

- 为了显得极简而删除用户完成任务所需的信息。
- 保留大量重复信息，只是把它们换成更小的字号或更浅的颜色。

## 审查问题

**如果移除这个元素，用户的理解是否会变差？**
