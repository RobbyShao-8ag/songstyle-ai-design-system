# 原则优先级与冲突裁决

SongStyle 的原则不是装饰偏好，也不是把页面统一压低、留空或古典化。当可用性、真实性、必要密度与审美表达发生冲突时，先判断约束层级，再决定如何取舍。

## 分层模型

1. 硬约束：法律合规、安全、可访问性、核心业务流程和关键用户任务不能被牺牲。任何宋式表达都不能破坏这些条件。
2. 任务目标：页面首先要帮助用户完成当下任务，例如理解价值、比较方案、提交表单、完成交易或找到证据。
3. 信息秩序：在必要信息完整的前提下，建立主次、节奏、分组与阅读路径，让复杂内容仍然清楚。
4. 审美表达：色彩、空间、动效、图形和语气负责传达温润、克制与余韵，但必须服务前面三层。

如果上一层尚未满足，下一层不能独立作为决策理由。低饱和、留白或含蓄表达只有在不损害硬约束、任务目标和信息秩序时才成立。

## 冲突裁决

- 可用性与审美表达冲突时，保留可理解、可操作、可反馈的界面，再降低表达强度。
- 真实性与品牌气质冲突时，保留真实证据、数据边界和来源说明，再调整语气与呈现节奏。
- 必要密度与留白冲突时，保留必要内容，通过分组、渐进展开、表格或摘要改善秩序。
- 信息秩序与视觉均衡冲突时，优先让用户看见决策路径，而不是追求静态构图的空灵。
- 任务目标与含蓄表达冲突时，关键行动、价格、风险、限制和下一步必须明确。

裁决结果需要记录为可复查的决定：保留什么、删除什么、延后什么、合并什么，以及这样做如何支持任务目标。

## 审查问题

- 这个页面是否先满足硬约束，再讨论宋式气质？
- 用户的任务目标是否清楚，关键行动是否可见？
- 信息秩序是否帮助用户判断轻重，而不是单纯减少内容？
- 审美表达是否影响了可读性、可操作性或信任证据？
- 每个冲突裁决是否说明了被保护的上层约束？
