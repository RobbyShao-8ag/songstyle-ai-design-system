# 信息距离

信息距离把三远法转译为现代 Web 与界面设计的层级方法。它不要求页面模拟山水画，而是帮助设计者安排用户先看什么、接着理解什么、最后如何形成整体判断。

## 近景

近景是用户当前必须处理的信息：主标题、关键价值、核心行动、输入焦点、错误提示、价格、状态和风险。近景需要明确、可读、可操作，不能为了含蓄而隐藏必要信息。

## 中景

中景解释近景的理由和结构：功能分组、方案比较、步骤说明、证据摘要、使用场景和辅助行动。中景负责把信息秩序展开，让用户知道为什么当前行动值得继续。

## 远景

远景提供背景、余韵和长期信任：品牌主张、方法论、案例脉络、价值观、延伸阅读和完整证据。远景可以更克制、更留白，但仍然要能被找到和理解。

## 应用规则

- 先确定任务链路中的近景，再安排中景解释和远景支撑。
- 同一屏内不要让多个近景互相争夺注意力。
- 中景可以通过卡片、分区、摘要或对比表组织复杂内容。
- 远景不等于废弃信息；它是低频但有价值的判断依据。
- 当三远法与可用性冲突时，优先保持信息距离清楚，而不是追求形式隐喻。

## 审查问题

- 用户能否在近景中立即找到当前任务、核心行动和关键反馈？
- 中景是否解释了选择理由、结构关系和下一步路径？
- 远景是否提供背景和信任，而不是堆叠装饰性叙事？
- 信息距离是否让复杂页面更清楚，而不是把内容藏得更深？
