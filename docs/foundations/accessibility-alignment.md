# 可访问性对齐

宋式表达可以低饱和、温润、克制，但必须温润但合规。可访问性不是风格之外的检查项，而是硬约束：用户必须能阅读、聚焦、操作、理解状态，并在失败时得到明确反馈。

## 温润但合规

温润不等于低对比，克制不等于弱反馈。背景、文字、边框、图标、焦点状态和错误状态都要在真实使用环境中保持清楚。低饱和色可以用于情绪与层次，但不能让关键文字、按钮或状态难以辨认。

## WCAG 使用边界

- 使用 WCAG 对比度要求作为最低检查线，尤其是正文、说明文字、按钮文字、表单标签和错误信息。
- 不把颜色作为唯一的信息传达方式，状态变化需要文本、形状、图标或位置辅助。
- 焦点状态必须可见，键盘用户应能确认当前位置和可操作目标。
- 动效应服务理解，不制造眩晕、延迟或不可预测的操作结果。
- 对比度通过后仍需人工检查实际场景：浅色背景、透明叠层、禁用态和小字号都可能降低可读性。

## 审查问题

- 关键文字和控件是否满足 WCAG 对比度的最低要求？
- 焦点状态是否清楚可见，并且不只依赖细微色差？
- 低饱和表达是否保留了足够的层级、边界和操作反馈？
- 错误、警告、成功和禁用状态是否能被色弱或键盘用户理解？
- 页面是否在保持宋式温润感的同时，仍然让用户清楚完成任务？
