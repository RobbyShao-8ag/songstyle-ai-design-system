# 表面

## 设计决策

表面应支持内容关系，而不是把每个模块塑造成独立物体。优先弱边界、柔和阴影与低塑料感。

## 相关设计变量

参见 [Design Tokens / 设计变量](../../design-tokens/) 中的 `shape.*` 与 `color.surface.*`。

## 适合使用

用细边界区分卡片，用轻微阴影表达少量层级。

## 不鼓励的模式

避免厚重浮雕、过度圆角、强玻璃拟态和每个区域都悬浮的布局。

## 审查问题

表面是否帮助层级，而没有抢走内容注意力？
