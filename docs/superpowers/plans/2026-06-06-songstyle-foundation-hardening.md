# 原则优先级与冲突裁决

六项原则不是简单并列的装饰偏好。真正的使用方式是先确认不能被破坏的硬约束，再决定页面目标、信息秩序和审美表达。

## 分层模型

1. **硬约束**：可访问性、中文标题可读性、必要内容、真实表述、技术可行性、法律与业务要求不能被审美偏好覆盖。
2. **任务目标**：页面首先要帮助用户完成当前任务，品牌表达不能让主任务失焦。
3. **信息秩序**：用近景、中景、远景组织内容，让必要信息有清楚关系。
4. **审美表达**：留白、克制、温润、含蓄和余韵只能强化上面三层，不能替代它们。

## 冲突裁决

| 冲突 | 裁决规则 |
| --- | --- |
| 留白生意 vs 必要密度 | 保留必要信息，改用分组、节奏和层级创造呼吸。 |
| 含蓄留韵 vs 清楚说明 | 必要任务、价格、风险、来源和操作结果必须说明清楚。 |
| 温润可亲 vs 对比度 | 使用高对比文本和清晰焦点状态，不能用低对比伪装温润。 |
| 克制有度 vs 转化目标 | CTA 和信任证据必须可见，克制的是噪音，不是行动路径。 |
| 秩序而非符号 vs 文化表达 | 优先用比例、节奏、材质和关系表达文化气质，符号必须服务功能。 |
| 标题构图 vs 中文可读性 | 保留语义完整和舒适行宽，不能为了留白制造破碎标题。 |

## 审查问题

- 这里是否有硬约束被审美选择覆盖？
- 用户任务是否仍然可以顺利完成？
- 被删除、延后或弱化的信息是否仍然可追溯？
- 留白是否在建立关系，而不是制造空感？
```

# 信息距离

SongStyle 使用“三远法”不是为了模拟山水画，而是为了建立信息距离。界面中的近景、中景和远景决定用户先看什么、如何相信、在哪里停留。

## 近景

近景是用户立即要理解或执行的内容：主标题、核心价值、主 CTA、关键状态、产品主体或当前任务。近景应在首屏早段出现，并拥有最清楚的层级。

## 中景

中景提供支持：特性、规格、引用、证据、流程关系、反对意见处理和次级操作。中景应帮助近景成立，但不能和近景争夺第一注意力。

## 远景

远景承载品牌故事、长期愿景、氛围、哲学、背景和可延后探索的内容。远景可以更含蓄，但不能隐藏必要信息。

## 应用规则

- 产品落地页：近景是价值与行动，中景是产品事实和信任证据，远景是品牌气质。
- 内容阅读页：近景是标题与来源，中景是结构和论据，远景是相关背景与延伸阅读。
- 数据或专家工具：近景是当前任务和状态，中景是数据关系和控件，远景是趋势、历史和解释。
- 交易页面：近景是商品、价格、风险和确认动作，中景是条款、支持和校验，远景只承担辅助信任。

## 审查问题

- 首屏近景是否清楚？
- 中景是否在提供证据，而不是制造噪音？
- 远景是否留出余韵，同时不遮蔽必要事实？
```

# 可访问性对齐

温润不是低对比。SongStyle 可以使用低饱和色、柔和表面和安静节奏，但正文、行动、焦点、错误和必要信息必须清楚可见。

## 温润但合规

- 正文优先使用 `color.text.primary`，在浅色背景上保持稳定对比度。
- `color.text.secondary` 用于支持信息，不能承载唯一的关键操作。
- `color.text.muted` 只用于低优先级辅助内容，不用于价格、错误、风险、法律或主 CTA。
- 主 CTA、链接、焦点状态和错误状态必须有可辨认的颜色、边框、位置或文本标签。
- 装饰性低对比表面不能替代控件边界和状态反馈。

## WCAG 使用边界

实现页面时，正文和关键控件应满足 WCAG AA；长正文、重要说明和交易信息应尽量接近 AAA。若品牌色过浅，应加深文本、增加边框或改变背景，而不是牺牲可读性。

## 审查问题

- 低饱和色是否仍然让信息清楚？
- 用户能否识别焦点、错误、可点击状态和主要行动？
- 温润感是否来自材质、节奏和比例，而不是来自降低可读性？

# 必要密度边界

更空不等于更好。SongStyle 可以删除噪音、合并重复、延后次要内容，但不能隐藏必要信息。

## 页面类型下界

| 页面类型 | 必须保留 |
| --- | --- |
| 品牌或活动页 | 价值主张、受众相关性、可信证据、下一步行动。 |
| 产品落地页 | 价值主张、产品事实、价格或购买路径、信任证据、主 CTA。 |
| 内容阅读页 | 标题、来源、日期或时效标记、结构、导航和舒适阅读节奏。 |
| 数据或专家工具 | 当前任务、状态、控件、反馈、密集但有序的数据、错误恢复。 |
| 交易页面 | 商品或服务、价格、条款、风险、确认动作、支持路径。 |

## 决策类型

- **删除**：内容不支持用户任务、业务目标或可信理解。
- **延后**：内容有价值，但不属于近景任务。
- **合并**：重复内容表达同一含义，可以收束。
- **保留**：内容属于硬约束或必要任务路径。
- **澄清**：内容必要但表达含糊，需要重写而不是删除。

## 审查问题

- 删除后，用户是否仍能完成任务？
- 延后后，必要信息是否仍然可见和可达？
- 这里的留白是否以牺牲理解为代价？

# 常见误解

## SongStyle 不是极简

极简常以减少元素为表面结果。SongStyle 关注的是关系是否准确：必要信息可以密集，但必须有秩序。

## SongStyle 不是古风

宋式转译不要求页面历史化。文化气质来自比例、节奏、材料感、平静叙事和适度停止。

## SongStyle 不是删减信息

可以删除噪音，不能删除必要内容。产品事实、主 CTA、风险、来源、价格和任务状态必须清楚。

## SongStyle 不是低对比

温润不等于看不清。低饱和表面必须配合清楚文本、焦点、错误和操作反馈。

## SongStyle 不是水墨符号库

水墨、印章、传统纹样和书法字体不是默认答案。若使用符号，它必须承担功能，而不是替代秩序。
```

# 反模式

## 空白崇拜

问题：大面积空白没有聚焦、分组、节奏、呼吸或联想功能。
修正：说明这块空间服务哪个信息关系；若说不清，就收紧空间或补回必要内容。

## 标题破碎

问题：为了留白把中文标题压成窄列，出现单字悬挂或破碎语义。
修正：扩大标题容器、调整字号和行高，让语义短语保持完整。

## 柔和但不可访问

问题：用低对比文字、模糊边界或弱焦点状态制造温润感。
修正：提高文本和控件对比度，用材质、节奏和比例表达温润。

## 文化符号堆砌

问题：水墨、印章、纹样、书法字体成为主要风格来源。
修正：先用信息秩序、留白、比例、色调和材料感建立气质。

## 过度克制

问题：主 CTA、证据、价格、风险或任务反馈太弱，用户不知道下一步。
修正：保留清楚行动路径，把克制用于减少噪音，而不是减少可用性。

## AI 过量生成

问题：渐变、徽章、图标、卖点卡片和重复段落持续堆叠。
修正：每个新增元素必须说明任务、业务、可访问性或意义功能；说不清就停止添加。

function renderList(label, items = []) {
  if (!items.length) return "";
  return `- ${label}:\n${items.map((item) => `  - ${item}`).join("\n")}`;
}

function renderScoreRubric(scoreRubric = {}) {
  const scores = ["0", "1", "2", "3", "4"];
  return `- Score rubric:\n${scores
    .map((score) => `  - ${score}: ${scoreRubric[score]}`)
    .join("\n")}`;
}
```
