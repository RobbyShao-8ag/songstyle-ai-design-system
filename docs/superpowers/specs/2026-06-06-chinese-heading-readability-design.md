# Chinese Heading Readability Design

## Goal

Fix the current SongStyle mobile examples where large Chinese headings break
into awkward fragments, and strengthen the design system so future SongStyle
work treats readability as a hard constraint, not a preference that can be
overridden by whitespace.

## Problem

The current Baiting and Qingxu SongStyle mobile treatments use very narrow
heading widths (`10ch` and `11ch`) with large display type. This creates forced
line breaks such as single-character endings and broken semantic phrases. The
result looks sparse, but the reading rhythm is visibly worse.

This is not a Song aesthetics requirement. Existing project guidance already
says Chinese readability comes first and that title breaks should feel
composed. The gap is that those principles are not expressed as concrete
implementation and review rules.

## Design Principle

SongStyle heading composition must follow this order:

1. Preserve semantic reading units.
2. Maintain comfortable Chinese line rhythm.
3. Use whitespace to frame and pace the message.

Whitespace may shape a title, but it may not create broken reading.

## Implementation Scope

### Current Examples

Update:

- `website/src/components/examples/BaitingSongStyleMobile.astro`
- `website/src/components/examples/QingxuSongStyleMobile.astro`

The headings should:

- Avoid forced ultra-narrow `ch` columns.
- Use a readable display size inside the phone frame.
- Keep the current two example headlines within 2 to 3 composed lines at the
  desktop comparison viewport.
- Avoid orphaned single Chinese characters at the end of a line.
- Preserve the SongStyle calm, editorial tone.

No case copy, images, CTAs, or comparison data should change.

### Design System Rules

Update the public design guidance:

- `docs/foundations/typography.md`
- `docs/references/song-aesthetics-digital-design-research.md`
- `checklists/songstyle-review.md`
- `prompts/web-design/from-brief.md`
- `prompts/web-design/rewrite-existing-page.md`
- `prompts/design-review/review-page.md`
- `prompts/design-review/compare-default-and-songstyle.md`

Add a rule that Chinese headings must not be made narrow just to create
whitespace. Reviews should flag:

- single-character line endings in display headings;
- broken semantic phrases;
- decorative narrow columns that reduce comprehension;
- large display type that forces avoidable fragmentation.

## Verification

Add regression coverage that asserts the Baiting and Qingxu SongStyle headings
use a readable width and do not end any rendered line with a single Chinese
character.

The test should use browser-side range measurement to inspect rendered Chinese
heading lines at the existing desktop comparison viewport. It should fail
against the current implementation and pass after the layout fix.

Run:

- focused Playwright test for the two case pages;
- `npx astro check`;
- `npm run verify`.

## Non-Goals

- No rewrite of the case-study copy.
- No new visual style direction.
- No reduction of SongStyle whitespace as a principle.
- No broad typography token redesign in this pass.
