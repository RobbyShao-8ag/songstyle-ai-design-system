# SongStyle AI Design System Roadmap

SongStyle is a creator-led open-source project. The Roadmap prioritizes proof
and operational clarity before breadth.

## Shipped: v0.1 Foundation

- Chinese manifesto, principles, foundations, guides, and review checklist
- Concise English entry documentation
- Machine-readable Design Tokens and generated CSS
- Web design and design review Prompts
- Two portable Agent Skills
- Reference UI, static documentation website, and two fictional-brand cases
- Automated unit, build, link, accessibility, desktop, and mobile checks

## Now: Evidence And Community Validation

- Make the homepage and README explain the value through fair visual evidence
- Rebuild cases with the same copy, image, features, and goal across treatments
- Invite designers and AI-assisted developers into the first user study
- Collect community usage feedback and case-study proposals
- Identify where SongStyle is misunderstood as minimalism or ancient style

## v0.2 Focus: Operational Clarity

- Publish results from real Prompt and Agent Skill use
- Add community cases across SaaS, commerce, content, and data-dense products
- Refine Design Token usage guidance and necessary-density examples
- Define a structured design review result schema
- Improve Chinese-English content parity
- Clarify how SongStyle adapts to different brands without becoming one skin

## Later Possibilities, Pending Evidence

- Additional interface patterns
- Broader mobile and brand-visual guidance
- Design-tool integrations
- Machine-readable review integrations

These are not commitments. SongStyle will not add a production component
library, online generator, Figma plugin, or MCP server until real use proves
that such work serves the design thesis.

## How To Influence The Roadmap

- Submit structured usage feedback through the GitHub issue form.
- Propose a community case study with the original brief and evidence.
- Join the [first-round user testing plan](docs/research/first-round-user-testing.md).
- Open a design discussion when a principle or tradeoff needs clarification.
